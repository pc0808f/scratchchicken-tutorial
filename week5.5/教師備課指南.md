## 🎯 備課步驟（按順序執行）

### 第一步：理解課程內容（30-45 分鐘）

**1.1 閱讀課程頁面**
- 詳細閱讀 [week5.5/index.md](./index.md)
- 理解每個部分的教學重點
- 記下可能的學生困難點

**1.2 理解技能整合**
- 複習第 1-5 週的關鍵技能
- 理解每個技能在遊戲中的應用
- 準備回答學生可能的問題

**重點技能對照**：
```
造型切換（第2週） → 跑步動畫
事件觸發（第2週） → 跳躍控制
變數（第3週）     → 計分系統
廣播（第3週）     → 遊戲結束通知
條件判斷（第4週） → 碰撞偵測
分身（第5週）     → 障礙物產生
隨機（第5週）     → 障礙物間隔
```

### 第二步：製作展示範例專案（60-90 分鐘）

**目的**：讓學生看到完成品，提升學習動機

**2.1 開啟新專案**
1. 進入 Scratch 3.0
2. 建立新專案
3. 命名：「小電子雞跑酷-展示版」

**2.2 準備角色**

**電子雞角色**：
1. 選擇角色：
   - 使用內建「Chick」或「Duck」
   - 或上傳自己設計的電子雞

2. 建立跑步造型：
   ```
   造型1：左腳在前
   造型2：右腳在前
   ```

   **快速方法**：
   - 複製原始造型 2 次
   - 使用 Scratch 的翻轉/旋轉工具
   - 或直接修改腿部位置

3. 建立其他造型（選做）：
   - 跳躍造型
   - 遊戲結束造型

**障礙物角色**：
1. 選擇 1-2 種障礙物：
   - 仙人掌（Cactus）
   - 石頭（Rocks）
   - 或自己畫

2. 調整大小：
   - 縮小到 40-60%
   - 確保電子雞能跳過

**地面角色**（選做）：
1. 畫一條長長的地面線
2. 或使用背景的地平線

**2.3 依序完成功能**

**建議順序**（重要！跟著這個順序做）：

```
1. 跑步動畫（5分鐘）
   ↓
2. 簡單跳躍（10分鐘）
   ↓ 測試：能跑能跳
3. 障礙物產生（15分鐘）
   ↓ 測試：障礙物會出現並移動
4. 碰撞偵測（10分鐘）
   ↓ 測試：碰到會遊戲結束
5. 計分系統（10分鐘）
   ↓ 測試：分數會增加
6. 重新開始（10分鐘）
   ↓ 測試：可以重新開始
7. 音效與優化（20分鐘）
   ↓
完成！
```

**2.4 測試遊戲**
- 完整玩一遍遊戲
- 確認所有功能正常
- 記錄可能的 bug 和解決方法

**2.5 儲存專案**
- 點擊「檔案」→「儲存到你的電腦」
- 命名：`第五點五週-完整範例.sb3`
- 備份到多個位置

### 第三步：製作教學用專案（30-45 分鐘）

**目的**：上課時跟著學生一起從零開始做

**3.1 建立空白專案**
1. 新增專案
2. 命名：「小電子雞跑酷-教學版」

**3.2 只準備基礎設定**
- 加入電子雞角色（已有跑步造型）
- 加入障礙物角色（已調整好大小）
- 設定簡單的背景
- **不要加任何程式碼**

**3.3 儲存專案**
- 儲存為：`第五點五週-教學空白版.sb3`

**為什麼要準備空白版？**
- 節省上課時間（不用花時間畫造型）
- 專注在程式邏輯教學
- 確保所有學生使用相同的素材

### 第四步：準備分階段檢查點（30 分鐘）

**目的**：讓學生可以分段完成，避免一次做不完

**4.1 建立檢查點專案**

依照課程進度，建立 4 個檢查點專案：

**檢查點 1：跑步動畫完成**
```
功能：
✅ 電子雞會在原地跑步
✅ 造型會自動切換

測試方法：
按綠旗，看到電子雞跑步動畫
```
儲存為：`檢查點1-跑步動畫.sb3`

**檢查點 2：加入跳躍**
```
功能：
✅ 檢查點 1 的功能
✅ 按空白鍵或點擊可以跳躍
✅ 跳躍後會回到地面

測試方法：
按空白鍵，電子雞會跳起來再落下
```
儲存為：`檢查點2-跳躍系統.sb3`

**檢查點 3：障礙物出現**
```
功能：
✅ 檢查點 2 的功能
✅ 障礙物會從右側出現
✅ 障礙物會向左移動
✅ 障礙物會持續產生

測試方法：
等待幾秒，看到障礙物持續出現並移動
```
儲存為：`檢查點3-障礙物系統.sb3`

**檢查點 4：完整遊戲**
```
功能：
✅ 檢查點 3 的功能
✅ 碰到障礙物會遊戲結束
✅ 分數會持續增加
✅ 可以重新開始

測試方法：
完整玩一局遊戲
```
儲存為：`檢查點4-完整遊戲.sb3`

**4.2 為什麼需要檢查點？**
- 學生進度不同時，可以給落後的學生載入檢查點
- 學生做錯時，可以回到上一個檢查點重新開始
- 減少學生的挫折感

**5.2 關鍵程式碼投影片**

**跳躍程式碼**：
```scratch
當 [空白 v] 鍵被按下
如果 <(y座標) = (地面高度)> 那麼
  重複 [10] 次
    y 改變 [15]        ← 往上跳
    等待 [0.03] 秒
  結束
  重複 [10] 次
    y 改變 [-15]       ← 落下
    等待 [0.03] 秒
  結束
  y 座標設為 (地面高度)
結束
```

**障礙物分身程式**：
```scratch
當分身產生
顯示
x 座標設為 [250]
重複無限次
  x 改變 [-10]        ← 向左移動
  如果 <(x座標) < [-250]> 那麼
    刪除此分身
  結束
結束
```

**5.3 除錯提示投影片**

```
常見問題與解決方法：

❌ 電子雞可以無限跳
✅ 加入「如果 y座標 = 地面高度」

❌ 障礙物不會移動
✅ 檢查「x 改變」是否在重複無限次裡

❌ 跳躍後飛到天上
✅ 確保往上和往下的重複次數相同

❌ 遊戲結束後障礙物還在跑
✅ 加入遊戲狀態判斷
```

## 🆘 緊急應變方案

### 方案 1：時間不夠

**如果 75 分鐘無法完成**：

**優先順序**：
1. ✅ 必須完成：跑步 + 跳躍
2. ⭐ 重要：障礙物產生
3. 🌟 進階：碰撞與計分

**調整策略**：
- 跳過音效部分
- 簡化跳躍程式（只用基本版）
- 提供檢查點 3 讓學生載入
- **重新開始功能使用方法三**（提示按綠旗）

{: .important }
> **💡 關於「重新開始」功能的教學選擇**
>
> **方法一/二（完整版）**：
> - ✅ 適合時間充裕的課堂
> - ✅ 學生能學習遊戲狀態管理
> - ✅ 使用者體驗更好
> - ❌ 需要額外 10-15 分鐘教學
>
> **方法三（簡化版）**：
> - ✅ 只需 2 分鐘教學
> - ✅ 不會出錯，容易理解
> - ✅ 適合時間緊迫的情況
> - ❌ 使用者體驗較差
> - ❌ 失去學習機會
>
> **建議**：
> - 第一次教這堂課 → 使用方法三
> - 熟悉課程流程後 → 使用方法一或二

### 方案 2：學生普遍跟不上

**症狀**：超過一半的學生卡在某個步驟

**處理方式**：
1. 暫停所有學生
2. 全班一起看投影幕
3. 老師重新示範一次（慢速）
4. 請一位完成的學生分享步驟
5. 給予 5 分鐘實作時間
6. 再次檢查

### 方案 3：電腦技術問題

**Scratch 無法開啟**：
- 切換到離線版 Scratch
- 或改用其他電腦

**專案檔案損壞**：
- 使用備份的檢查點專案
- 或使用雲端備份

**網路斷線**：
- 切換到離線版
- 提醒學生先下載專案

### 方案 4：學生完全不懂跳躍原理

**簡化教學法**：

1. **生活化比喻**：
   「你跳起來的時候，會發生什麼？」
   「會先往上，然後往下對不對？」

2. **數線解釋**：
   在黑板畫一條數線：
   ```
   -100 (地面) → 0 (往上) → 50 (最高點) → 0 → -100 (回到地面)
   ```

3. **動作示範**：
   請一位學生跳一下，全班觀察
   「看！他先往上，到最高點，再落下」

4. **程式對應**：
   ```
   往上 = y 改變 15（做 10 次）
   落下 = y 改變 -15（做 10 次）
   ```

## 💡 教學小技巧

### 技巧 1：善用「暫停-展示-繼續」

```
1. 暫停：「大家先暫停，看這裡」
2. 展示：示範或說明
3. 繼續：「好，現在繼續做」
```

### 技巧 2：設定小目標

```
「現在給你們 5 分鐘，完成跑步動畫」
（5 分鐘後檢查）
「完成的舉手！」
```

### 技巧 3：同儕教學

```
「已經完成的同學，請幫助旁邊還沒完成的同學」
→ 強化強的學生的理解
→ 幫助弱的學生理解
```

### 技巧 4：及時獎勵

```
準備小獎勵：
• 完成跑步動畫 → 「很好！」貼紙
• 完成跳躍系統 → 「太棒了！」貼紙
• 完成整個遊戲 → 「程式高手」貼紙
```

### 技巧 5：問題引導

```
學生：「老師，我的不會跳！」
老師：「讓我們一起看看，你按空白鍵了嗎？」
學生：「按了」
老師：「那我們檢查一下，『當空白鍵被按下』有沒有寫？」
→ 引導學生自己找出問題
```
