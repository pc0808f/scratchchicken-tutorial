# ✅ 第 5.5 週快速備課檢查表

## 🎯 備課三步驟（最少需要 2 小時）

### 第一步：製作展示範例（60-90 分鐘）⭐ 最重要

**目標**：做一個完整的遊戲給學生看

**步驟**：
1. [ ] 開啟 Scratch，新增專案
2. [ ] 加入電子雞角色（Chick 或 Duck）
3. [ ] 建立 2 個跑步造型（複製原造型，修改腿的位置）
4. [ ] 加入障礙物角色（Cactus，縮小到 50%）
5. [ ] 依序完成功能：
   - [ ] 跑步動畫（10分鐘）
   - [ ] 跳躍系統（15分鐘）
   - [ ] 障礙物產生與移動（20分鐘）
   - [ ] 碰撞偵測（10分鐘）
   - [ ] 計分系統（10分鐘）
   - [ ] 遊戲結束與重新開始（15分鐘）
6. [ ] 測試遊戲，確認正常運作
7. [ ] 儲存為：`第五點五週-完整範例.sb3`

**重要**：這個範例是上課時給學生看的，要確保能正常運作！

---

### 第二步：準備教學用空白版（15 分鐘）

**目標**：節省上課時間，讓學生專注在程式

**步驟**：
1. [ ] 新增專案
2. [ ] 加入電子雞角色（已有 2 個跑步造型）
3. [ ] 加入障礙物角色（已調整好大小）
4. [ ] 不要加任何程式碼
5. [ ] 儲存為：`第五點五週-教學空白版.sb3`
6. [ ] 上傳到學生可以下載的地方

---

### 第三步：準備檢查點專案（30-45 分鐘）⭐ 建議做

**目標**：學生進度不同時可以使用

**需要準備的檢查點**：

**檢查點 1**：跑步動畫完成
- [ ] 複製展示範例
- [ ] 只保留跑步動畫程式
- [ ] 刪除其他功能
- [ ] 儲存為：`檢查點1-跑步動畫.sb3`

**檢查點 2**：跳躍系統完成
- [ ] 複製展示範例
- [ ] 保留跑步 + 跳躍程式
- [ ] 刪除障礙物等功能
- [ ] 儲存為：`檢查點2-跳躍系統.sb3`

**檢查點 3**：障礙物完成
- [ ] 複製展示範例
- [ ] 保留跑步 + 跳躍 + 障礙物
- [ ] 刪除碰撞偵測等功能
- [ ] 儲存為：`檢查點3-障礙物系統.sb3`

**檢查點 4**：完整遊戲
- [ ] 就是展示範例
- [ ] 儲存為：`檢查點4-完整遊戲.sb3`

## 🆘 緊急應變

### 時間不夠
**症狀**：75 分鐘無法完成

**處理**：
- 優先完成：跑步 + 跳躍
- 其他功能下週繼續
- 或提供檢查點 3 讓學生載入

### 學生普遍卡住
**症狀**：超過一半學生卡在某步驟

**處理**：
1. 暫停所有學生
2. 全班一起看投影
3. 重新示範一次（慢速）
4. 再給 5 分鐘實作

### 學生不懂跳躍
**症狀**：學生無法理解跳躍原理

**處理**：
1. 生活化：「你跳起來會怎樣？先往上再往下對吧」
2. 畫數線：在黑板畫出 y 座標變化
3. 請一位學生跳一下，全班觀察
4. 對應到程式：往上 = y 改變 15，往下 = y 改變 -15

## 💡 備課重點提醒

### 三個關鍵
1. **展示範例最重要** - 學生要先看到成果才有動力
2. **分段教學** - 不要一次教完，要分 4 個部分
3. **控制時間** - 每個部分嚴格控時，避免拖延

### 常見錯誤
❌ 沒有準備展示範例 → 學生不知道要做什麼
❌ 一次教太多 → 學生消化不良
❌ 沒有檢查點 → 進度慢的學生放棄
❌ 沒有測試環境 → 上課當天電腦出問題

### 成功秘訣
✅ 事前把遊戲做一遍 → 知道會遇到什麼問題
✅ 準備檢查點專案 → 學生可以分段完成
✅ 設定時間限制 → 避免某部分拖太久
✅ 保持熱情 → 感染學生的學習動力
